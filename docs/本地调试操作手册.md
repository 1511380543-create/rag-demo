# RAG 本地调试操作手册

## 1. 前置准备

- 本机环境：macOS（已适配）
- Python 建议版本：`3.10+`（推荐 conda 环境 `rag-demo`，当前为 Python 3.11）
- 项目根目录：`/Applications/pythonWorkSpace/rag-demo`
- 必需环境变量：`API_KEY_ALI`、`MYSQL_HOST`、`MYSQL_PORT`、`MYSQL_USER`、`MYSQL_PASSWORD`、`MYSQL_DATABASE`

## 2. 安装依赖

使用 conda 环境 `rag-demo`：

```bash
conda activate rag-demo
pip install -r requirements.txt
```

> 说明：项目目录下若存在 `.venv`，可能会覆盖 conda 的 `python`/`pip`。调试时建议确认当前解释器：
>
> ```bash
> which python
> # 期望输出类似：/Users/<user>/miniconda3/envs/rag-demo/bin/python
> ```

依赖分工：

- **PDF 抽取**：`unstructured==0.18.32` + `unstructured-inference>=1.5.2,<1.7`（推荐 `1.6.11`）
- **切块 / 索引**：`llama-index-core`（`SentenceSplitter`、向量索引）

版本说明：`unstructured 0.24.x` 需 `unstructured-inference>=1.6.12`；若 inference 控制在 1.5–1.6 段，请使用 `unstructured 0.18.32`。

PDF 解析相关包（如 `pdfminer.six`、`pdf2image`、`pillow`、`pillow-heif`、`poppler` 等）请在本机按需安装。

## 3. 启动服务

### 3.1 设置阿里云百炼 API Key

```bash
export API_KEY_ALI="你的阿里云百炼API Key"
```

### 3.2 设置 MySQL 连接信息

```bash
export MYSQL_HOST="127.0.0.1"
export MYSQL_PORT="3306"
export MYSQL_USER="root"
export MYSQL_PASSWORD="root"
export MYSQL_DATABASE="rag_demo"
```

### 3.3 初始化 MySQL 表结构

```bash
mysql -h127.0.0.1 -P3306 -uroot -proot < spec/sql/mysql_schema.sql
```

表结构包含 `rag_documents`（抽取结果）、`rag_chunks`（切块结果）等，详见 `spec/sql/mysql_schema.sql`。

### 3.4 启动 FastAPI 服务

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8080
```

开发调试场景如需自动热重载，可使用：

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8080
```

启动成功后可访问：

- Swagger 文档：`http://127.0.0.1:8080/docs`
- 健康检查：`http://127.0.0.1:8080/rag/health`

## 4. curl 调试命令

> 以下命令默认服务地址为 `http://127.0.0.1:8080`。

入库与检索的标准流程为三步：

```
POST /rag/extract  →  POST /rag/chunks  →  POST /rag/index/build  →  POST /rag/query
   抽取入库              切块入库              构建向量索引              在线召回
```

### 4.1 健康检查

```bash
curl -X GET "http://127.0.0.1:8080/rag/health"
```

响应字段说明：

- `extracted_docs`：MySQL 中已抽取文档数（`rag_documents`）
- `indexed_docs` / `indexed_chunks`：当前内存索引中的文档数与 chunk 数
- `index_ready`：索引是否可用于查询

### 4.2 文档抽取入库（`/rag/extract`）

从本地 PDF 抽取文本、表格（HTML）并写入 `rag_documents`：

```bash
curl -X POST "http://127.0.0.1:8080/rag/extract" \
  -H "Content-Type: application/json" \
  -d '{
    "documents": [
      {
        "doc_id": "pdf-obd-809",
        "file_path": "docs/OBD设备JT_T 809协议虚拟技术手册.pdf",
        "metadata": {
          "source": "local_pdf",
          "category": "policy"
        }
      }
    ]
  }'
```

批量抽取多份 PDF：

```bash
curl -X POST "http://127.0.0.1:8080/rag/extract" \
  -H "Content-Type: application/json" \
  -d '{
    "documents": [
      {
        "doc_id": "pdf-obd-809",
        "file_path": "docs/OBD设备JT_T 809协议虚拟技术手册.pdf",
        "metadata": {"source": "local_pdf", "category": "policy"}
      },
      {
        "doc_id": "pdf-emission-std",
        "file_path": "docs/国三排放标准柴油机动车报废管理实施标准（虚拟政策文档）.pdf",
        "metadata": {"source": "local_pdf", "category": "policy"}
      }
    ]
  }'
```

成功响应示例字段：`extracted_doc_count`、`total_page_count`、`total_char_count`、`reports`（逐文档清洗/表格统计）。

### 4.3 文档切块入库（`/rag/chunks`）

基于已抽取文档的 `full_text` 切块并写入 `rag_chunks`（须先完成 `/rag/extract`）：

```bash
curl -X POST "http://127.0.0.1:8080/rag/chunks" \
  -H "Content-Type: application/json" \
  -d '{
    "doc_ids": ["pdf-obd-809"]
  }'
```

批量切块：

```bash
curl -X POST "http://127.0.0.1:8080/rag/chunks" \
  -H "Content-Type: application/json" \
  -d '{
    "doc_ids": ["pdf-obd-809", "pdf-emission-std"]
  }'
```

### 4.4 索引构建（`/rag/index/build`）

```bash
curl -X POST "http://127.0.0.1:8080/rag/index/build" \
  -H "Content-Type: application/json" \
  -d '{
    "force_rebuild": true
  }'
```

如需仅构建指定文档：

```bash
curl -X POST "http://127.0.0.1:8080/rag/index/build" \
  -H "Content-Type: application/json" \
  -d '{
    "doc_ids": ["pdf-obd-809"],
    "force_rebuild": true
  }'
```

### 4.5 查询召回（`/rag/query`，默认 top_k=3）

```bash
curl -X POST "http://127.0.0.1:8080/rag/query" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "JT/T 809 协议默认通信端口和心跳周期是多少？"
  }'
```

### 4.6 查询召回（自定义 top_k 与 filters）

```bash
curl -X POST "http://127.0.0.1:8080/rag/query" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "OBD车辆故障码上报对应的报文ID是什么？",
    "top_k": 5,
    "filters": {
      "category": "policy"
    }
  }'
```

> 说明：`/rag/query` 为在线召回接口，结果写入监控日志表 `rag_query_logs`；不会自动进入测评表。

### 4.7 评测样本写入（`/rag/eval/dataset`）

前置条件：已完成 `/rag/extract`、`/rag/chunks` 与 `/rag/index/build`。

每条样本需提供 `relevant_chunk_ids`（chunk 级标注）或 `expected_keywords`（关键词标注）至少其一：

```bash
curl -X POST "http://127.0.0.1:8080/rag/eval/dataset" \
  -H "Content-Type: application/json" \
  -d '{
    "cases": [
      {
        "case_id": "eval-obd-port-heartbeat",
        "query_text": "JT/T 809 协议默认通信端口和心跳周期是多少？",
        "expected_keywords": ["11009", "30s", "心跳"],
        "top_k": 3
      },
      {
        "case_id": "eval-obd-fault-code",
        "query_text": "OBD车辆故障码上报对应的报文ID是什么？",
        "expected_keywords": ["0x6002", "故障码"],
        "top_k": 5
      },
      {
        "case_id": "eval-emission-scope",
        "query_text": "国三排放标准柴油机动车报废管理的适用范围是什么？",
        "expected_keywords": ["国三", "报废", "柴油"],
        "top_k": 5
      },
      {
        "case_id": "eval-heavy-diesel-compliance",
        "query_text": "重型柴油车免年检需要满足哪些合规条件？",
        "expected_keywords": ["免年检", "OBD", "合规"],
        "top_k": 5
      }
    ]
  }'
```

同 `case_id` 重复提交会覆盖更新。

### 4.8 评测样本列表（`/rag/eval/dataset`）

```bash
curl -X GET "http://127.0.0.1:8080/rag/eval/dataset"
```

### 4.9 执行测评（`/rag/eval/run`）

对评测集中 `enabled=true` 的样本执行一轮离线测评，结果写入 `rag_eval_runs` 与 `rag_eval_run_items`：

```bash
curl -X POST "http://127.0.0.1:8080/rag/eval/run" \
  -H "Content-Type: application/json" \
  -d '{
    "top_k": 5,
    "note": "三份PDF全量入库后的首轮测评"
  }'
```

仅测评指定样本：

```bash
curl -X POST "http://127.0.0.1:8080/rag/eval/run" \
  -H "Content-Type: application/json" \
  -d '{
    "case_ids": ["eval-obd-port-heartbeat", "eval-emission-scope"],
    "top_k": 3,
    "note": "部分样本回归"
  }'
```

### 4.10 评测历史（`/rag/eval/runs`）

```bash
curl -X GET "http://127.0.0.1:8080/rag/eval/runs?limit=10"
```

## 5. 常见问题排查

### 5.1 启动时报错：缺少 API_KEY_ALI

现象：服务启动失败，并提示缺少必要环境变量。  
处理：重新执行：

```bash
export API_KEY_ALI="你的阿里云百炼API Key"
```

### 5.2 查询时报错：INDEX_NOT_READY

现象：调用 `/rag/query` 返回 `400`，错误码 `INDEX_NOT_READY`。  
处理：按顺序调用 `/rag/extract`（抽取入库）、`/rag/chunks`（切块入库）与 `/rag/index/build`（构建索引），再执行查询。

### 5.3 切块时报错：DOCUMENT_NOT_EXTRACTED

现象：调用 `/rag/chunks` 返回 `400`，错误码 `DOCUMENT_NOT_EXTRACTED`。  
处理：先对对应 `doc_id` 调用 `/rag/extract` 完成抽取，再执行切块。

### 5.4 构建索引时报错：NO_CHUNKS_FOR_INDEX

现象：调用 `/rag/index/build` 返回 `400`，错误码 `NO_CHUNKS_FOR_INDEX`。  
处理：先调用 `/rag/chunks` 写入 chunk 数据，再重新构建索引。

### 5.5 参数校验失败（422）

常见原因：

- `/rag/extract`：`documents` 为空数组；`doc_id` 或 `file_path` 为空白；`file_path` 不是 `.pdf` 文件；本地文件不存在
- `/rag/chunks`：`doc_ids` 为空数组或含空字符串
- `/rag/query`：`query` 为空白；`top_k` 超出 `1-20`
- 评测样本 `cases` 为空数组；未提供 `relevant_chunk_ids` 与 `expected_keywords`

请按接口模型要求修正请求体后重试。

### 5.6 测评时报错：EVAL_DATASET_EMPTY

现象：调用 `/rag/eval/run` 返回 `400`，错误码 `EVAL_DATASET_EMPTY`。  
处理：先调用 `/rag/eval/dataset` 写入评测样本，再执行测评。

### 5.7 测评时报错：INDEX_NOT_READY

现象：调用 `/rag/eval/run` 返回 `400`，错误码 `INDEX_NOT_READY`。  
处理：先完成 `/rag/extract`、`/rag/chunks` 与 `/rag/index/build`，再执行测评。服务重启后需重新 build 索引。
