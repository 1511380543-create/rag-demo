# RAG 本地调试操作手册

## 1. 前置准备

- 本机环境：macOS（已适配）
- Python 建议版本：`3.10+`
- 项目根目录：`/Applications/pythonWorkSpace/rag-demo`
- 必需环境变量：`API_KEY_ALI`、`MYSQL_HOST`、`MYSQL_PORT`、`MYSQL_USER`、`MYSQL_PASSWORD`、`MYSQL_DATABASE`

## 2. 安装依赖

在项目根目录执行：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## 3. 启动服务

### 3.1 设置阿里云百炼 API Key

```bash
export API_KEY_ALI="你的阿里云百炼API Key"
```

### 3.2 设置 MySQL 连接信息

```bash
export MYSQL_HOST="127.0.0.1"
export MYSQL_PORT="3306"
export MYSQL_USER="root"
export MYSQL_PASSWORD="root"
export MYSQL_DATABASE="rag_demo"
```

### 3.3 初始化 MySQL 表结构

```bash
mysql -h127.0.0.1 -P3306 -uroot -proot < spec/sql/mysql_schema.sql
```

### 3.4 启动 FastAPI 服务

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8080
```

开发调试场景如需自动热重载，可使用：

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8080
```

启动成功后可访问：

- Swagger 文档：`http://127.0.0.1:8080/docs`
- 健康检查：`http://127.0.0.1:8080/rag/health`

## 4. curl 调试命令

> 以下命令默认服务地址为 `http://127.0.0.1:8080`。

### 4.1 健康检查

```bash
curl -X GET "http://127.0.0.1:8080/rag/health"
```

### 4.2 文档切分入库（`/rag/chunks`）

```bash
curl -X POST "http://127.0.0.1:8080/rag/chunks" \
  -H "Content-Type: application/json" \
  -d '{
    "documents": [
      {
        "doc_id": "pdf-obd-809",
        "file_path": "docs/OBD设备JT_T 809协议虚拟技术手册.pdf",
        "metadata": {
          "source": "local_pdf",
          "category": "policy"
        }
      }
    ]
  }'
```

### 4.3 索引构建（`/rag/index/build`）

```bash
curl -X POST "http://127.0.0.1:8080/rag/index/build" \
  -H "Content-Type: application/json" \
  -d '{
    "force_rebuild": true
  }'
```

如需仅构建指定文档：

```bash
curl -X POST "http://127.0.0.1:8080/rag/index/build" \
  -H "Content-Type: application/json" \
  -d '{
    "doc_ids": ["pdf-obd-809"],
    "force_rebuild": true
  }'
```

### 4.4 查询召回（`/rag/query`，默认 top_k=3）

```bash
curl -X POST "http://127.0.0.1:8080/rag/query" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "JT/T 809 协议默认通信端口和心跳周期是多少？"
  }'
```

### 4.5 查询召回（自定义 top_k 与 filters）

```bash
curl -X POST "http://127.0.0.1:8080/rag/query" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "OBD车辆故障码上报对应的报文ID是什么？",
    "top_k": 5,
    "filters": {
      "category": "policy"
    }
  }'
```

## 5. 常见问题排查

### 5.1 启动时报错：缺少 API_KEY_ALI

现象：服务启动失败，并提示缺少必要环境变量。  
处理：重新执行：

```bash
export API_KEY_ALI="你的阿里云百炼API Key"
```

### 5.2 查询时报错：INDEX_NOT_READY

现象：调用 `/rag/query` 返回 `400`，错误码 `INDEX_NOT_READY`。  
处理：先按顺序调用 `/rag/chunks`（切分入库）与 `/rag/index/build`（构建索引），再执行查询。

### 5.3 构建索引时报错：NO_CHUNKS_FOR_INDEX

现象：调用 `/rag/index/build` 返回 `400`，错误码 `NO_CHUNKS_FOR_INDEX`。  
处理：先调用 `/rag/chunks` 写入 chunk 数据，再重新构建索引。

### 5.4 参数校验失败（422）

常见原因：

- `documents` 为空数组
- `doc_id` 或 `file_path` 为空白
- `file_path` 不是 `.pdf` 文件
- `file_path` 指向的本地文件不存在
- `query` 为空白
- `top_k` 超出 `1-20`

请按接口模型要求修正请求体后重试。
