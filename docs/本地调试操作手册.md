# RAG 本地调试操作手册

## 1. 前置准备

- 本机环境：macOS（已适配）
- Python 建议版本：`3.10+`（推荐 conda 环境 `rag-demo`，当前为 Python 3.11）
- 项目根目录：`/Applications/pythonWorkSpace/rag-demo`
- 必需环境变量：`API_KEY_ALI`、`MYSQL_HOST`、`MYSQL_PORT`、`MYSQL_USER`、`MYSQL_PASSWORD`、`MYSQL_DATABASE`
- 可选环境变量：`RAG_MIN_SCORE`（检索最低相似度，默认 `0.5`；低于该分的候选不进入 `contexts`）

## 2. 安装依赖

使用 conda 环境 `rag-demo`：

```bash
conda activate rag-demo
python -m pip install -r requirements.txt
```

> 说明：项目目录下若存在 `.venv`，可能会覆盖 conda 的 `python`/`pip`。调试时建议确认当前解释器：
>
> ```bash
> which python
> # 期望输出类似：/Users/<user>/miniconda3/envs/rag-demo/bin/python
> ```

依赖分工（以 spec 为准）：

- **PDF 抽取（spec v0.7）**：本地 **MinerU**（`extract_version=mineru-v1`）；企业级清洗与表格门禁见 `spec/architecture/08_document_extraction.md`
- **切块 / 索引**：`llama-index-core`（`SentenceSplitter`、向量索引）

> **注意**：v0.7 抽取代码尚未落地前，运行中服务可能仍是旧版 Unstructured。开发完成后删除 Unstructured 依赖，并按 MinerU 官方文档安装模型/系统依赖。落地前请以 `08` 为准，勿继续按 Unstructured 扩功能。

## 3. 启动服务

### 3.1 设置阿里云百炼 API Key

```bash
export API_KEY_ALI="你的阿里云百炼API Key"
```

### 3.2 设置 MySQL 连接信息

```bash
export MYSQL_HOST="127.0.0.1"
export MYSQL_PORT="3306"
export MYSQL_USER="root"
export MYSQL_PASSWORD="root"
export MYSQL_DATABASE="rag_demo"
```

### 3.3 初始化 MySQL 表结构

```bash
mysql -h127.0.0.1 -P3306 -uroot -proot < spec/sql/mysql_schema.sql
```

表结构包含 `rag_documents`（抽取结果）、`rag_chunks`（切块结果）等，详见 `spec/sql/mysql_schema.sql`。

### 3.4 启动 FastAPI 服务

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8080
```

开发调试场景如需自动热重载，可使用：

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8080
```

启动成功后可访问：

- Swagger 文档：`http://127.0.0.1:8080/docs`
- 健康检查：`http://127.0.0.1:8080/rag/health`

## 4. curl 调试命令

> 以下命令默认服务地址为 `http://127.0.0.1:8080`。

入库与检索的标准流程为三步：

```
POST /rag/extract  →  POST /rag/chunks  →  POST /rag/index/build  →  POST /rag/query
   抽取入库              切块入库              构建向量索引              在线召回
```

### 4.1 健康检查

```bash
curl -X GET "http://127.0.0.1:8080/rag/health"
```

响应字段说明：

- `extracted_docs`：MySQL 中已抽取文档数（`rag_documents`）
- `indexed_docs` / `indexed_chunks`：当前内存索引中的文档数与 chunk 数
- `index_ready`：索引是否可用于查询

### 4.7 评测种子集导入（`/rag/eval/dataset`）

前置条件：十四份 PDF 已完成 `/rag/extract`、`/rag/chunks` 与 `/rag/index/build`。

官方种子集：`spec/eval/eval_dataset.json`（第一轮 50 + 第二轮出题约 32，合计以文件 `case_count` 为准；十四份 PDF）。文件含 `description` / `prerequisites` / `doc_ids`，接口只接受 `{ "cases": [...] }`。

> 已有库若缺列，先执行：
> `ALTER TABLE rag_eval_dataset ADD COLUMN expect_hit TINYINT UNSIGNED NOT NULL DEFAULT 1 COMMENT '期望命中:1正样本0负样本';`
> `ALTER TABLE rag_eval_dataset ADD COLUMN filters JSON NULL COMMENT '样本级元数据过滤';`

> **注意**：下面命令使用绝对路径，可在任意目录执行；若改用相对路径，必须先 `cd` 到项目根，否则会找不到 JSON，`-d` 为空并返回 `VALIDATION_ERROR`。

```bash
# 方式 A：jq（需已安装）
curl -X POST "http://127.0.0.1:8080/rag/eval/dataset" \
  -H "Content-Type: application/json" \
  -d "$(jq '{cases: .cases}' /Applications/pythonWorkSpace/rag-demo/spec/eval/eval_dataset.json)"

# 方式 B：无 jq 时用 Python
curl -X POST "http://127.0.0.1:8080/rag/eval/dataset" \
  -H "Content-Type: application/json" \
  -d "$(python -c 'import json; print(json.dumps({"cases": json.load(open("/Applications/pythonWorkSpace/rag-demo/spec/eval/eval_dataset.json"))["cases"]}, ensure_ascii=False))')"
```

期望响应：`{"upserted_count":82}`（以当前种子 `case_count` 为准）。同 `case_id` 重复提交会覆盖更新。

> 第二轮负样本含 `expect_hit=false`；filters 题含样本级 `filters`。政策文档须按 §4.2 第二轮 metadata（`category=policy` / `access_group=["ops"]`）入库后，filters 题才有意义。

也可手写单条/少量样本（每条须提供 `relevant_chunk_ids` / `expected_keywords` / `evidence_keys` 至少其一）：

```bash
curl -X POST "http://127.0.0.1:8080/rag/eval/dataset" \
  -H "Content-Type: application/json" \
  -d '{
    "cases": [
      {
        "case_id": "eval-obd-port-heartbeat",
        "query_text": "JT/T 809 协议默认通信端口和心跳周期是多少？",
        "expected_keywords": ["11009", "心跳周期"],
        "evidence_keys": [
          {"doc_id": "pdf-obd-809", "anchor_text": "11009"},
          {"doc_id": "pdf-obd-809", "anchor_text": "心跳周期"}
        ],
        "keyword_match_mode": "all",
        "top_k": 10
      }
    ]
  }'
```

### 4.9 切块冻结打版（`/rag/eval/chunk-freeze`）

从现行 `rag_chunks` 拷贝一版物理快照（不改变现行索引）。切块策略实质变更或标定门禁前建议打一版。

```bash
# 打版（标签须唯一）
curl -X POST "http://127.0.0.1:8080/rag/eval/chunk-freeze" \
  -H "Content-Type: application/json" \
  -d '{
    "freeze_label": "round2-baseline@2026-07-24",
    "note": "第二轮收官后切块冻结示例",
    "pipeline_version": "chunk-soft-max"
  }'

# 列表
curl -X GET "http://127.0.0.1:8080/rag/eval/chunk-freezes?limit=10"

# 详情（把 {freeze_id} 换成上一步返回的 id）
curl -X GET "http://127.0.0.1:8080/rag/eval/chunk-freezes/1"
```

> 空库打版返回 `400 NO_CHUNKS_FOR_FREEZE`；重复标签返回 `409 DUPLICATE_FREEZE_LABEL`。

### 4.10 执行测评（`/rag/eval/run`）

对评测集中 `enabled=true` 的样本执行一轮离线测评，结果写入 `rag_eval_runs` 与 `rag_eval_run_items`：

```bash
curl -X POST "http://127.0.0.1:8080/rag/eval/run" \
  -H "Content-Type: application/json" \
  -d '{
    "note": "十四份PDF全量入库后的测评"
  }'
```

仅测评指定样本：

```bash
curl -X POST "http://127.0.0.1:8080/rag/eval/run" \
  -H "Content-Type: application/json" \
  -d '{
    "case_ids": ["eval-obd-port-heartbeat", "eval-emission-scope"],
    "note": "部分样本回归"
  }'
```

> 说明：请求体不传 `top_k` 时，按各样本自身 `top_k` 执行（种子统一为 **10**）；传入请求级 `top_k` 会覆盖全部样本，**主 baseline 请勿覆盖为 5**。样本含 `evidence_keys` 时，测评前会映射到当期 chunk_id 再算指标。样本含 `filters` 时测评检索会启用元数据过滤；`expect_hit=false` 为负样本（top_k 内出现标注证据则判失败）。第一轮门禁 `run_id=8`（`06` §5.1）；第二轮全量门禁 `run_id=11`（`06` §5.2）；两轮不可直接混比。设计债见 `07` §3.5。

## 5. 常见问题排查

### 5.1 启动时报错：缺少 API_KEY_ALI

现象：服务启动失败，并提示缺少必要环境变量。  
处理：重新执行：

```bash
export API_KEY_ALI="你的阿里云百炼API Key"
```

### 5.2 查询时报错：INDEX_NOT_READY

现象：调用 `/rag/query` 返回 `400`，错误码 `INDEX_NOT_READY`。  
处理：按顺序调用 `/rag/extract`（抽取入库）、`/rag/chunks`（切块入库）与 `/rag/index/build`（构建索引），再执行查询。

### 5.3 切块时报错：DOCUMENT_NOT_EXTRACTED

现象：调用 `/rag/chunks` 返回 `400`，错误码 `DOCUMENT_NOT_EXTRACTED`。  
处理：先对对应 `doc_id` 调用 `/rag/extract` 完成抽取，再执行切块。

### 5.4 构建索引时报错：NO_CHUNKS_FOR_INDEX

现象：调用 `/rag/index/build` 返回 `400`，错误码 `NO_CHUNKS_FOR_INDEX`。  
处理：先调用 `/rag/chunks` 写入 chunk 数据，再重新构建索引。

### 5.5 参数校验失败（422）

常见原因：

- `/rag/extract`：`documents` 为空数组；`doc_id` 或 `file_path` 为空白；`file_path` 不是 `.pdf` 文件；本地文件不存在
- `/rag/chunks`：`doc_ids` 为空数组或含空字符串
- `/rag/query`：`query` 为空白；`top_k` 超出 `1-20`
- 评测样本 `cases` 为空数组；未提供 `relevant_chunk_ids` / `expected_keywords` / `evidence_keys`

请按接口模型要求修正请求体后重试。

### 5.6 测评时报错：EVAL_DATASET_EMPTY

现象：调用 `/rag/eval/run` 返回 `400`，错误码 `EVAL_DATASET_EMPTY`。  
处理：先按 §4.7 从 `spec/eval/eval_dataset.json` 导入种子集（或手写样本），再执行测评。

### 5.7 测评时报错：INDEX_NOT_READY

现象：调用 `/rag/eval/run` 返回 `400`，错误码 `INDEX_NOT_READY`。  
处理：先完成 `/rag/extract`、`/rag/chunks` 与 `/rag/index/build`，再执行测评。服务重启后需重新 build 索引。
