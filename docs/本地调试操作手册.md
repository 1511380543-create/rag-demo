# RAG 本地调试操作手册

## 1. 前置准备

- 本机环境：macOS（已适配）
- Python 建议版本：`3.10+`
- 项目根目录：`/Applications/pythonWorkSpace/rag-demo`
- 必需环境变量：`API_KEY_ALI`

## 2. 安装依赖

在项目根目录执行：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## 3. 启动服务

### 3.1 设置阿里云百炼 API Key

```bash
export API_KEY_ALI="你的阿里云百炼API Key"
```

### 3.2 启动 FastAPI 服务

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8080
```

开发调试场景如需自动热重载，可使用：

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8080
```

启动成功后可访问：

- Swagger 文档：`http://127.0.0.1:8080/docs`
- 健康检查：`http://127.0.0.1:8080/rag/health`

## 4. curl 调试命令

> 以下命令默认服务地址为 `http://127.0.0.1:8080`。

### 4.1 健康检查

```bash
curl -X GET "http://127.0.0.1:8080/rag/health"
```

### 4.2 单文件入库（`/rag/index`）

```bash
curl -X POST "http://127.0.0.1:8080/rag/index" \
  -H "Content-Type: application/json" \
  -d '{
    "documents": [
      {
        "doc_id": "pdf-obd-809",
        "file_path": "docs/OBD设备JT_T 809协议虚拟技术手册.pdf",
        "metadata": {
          "source": "local_pdf",
          "category": "policy"
        }
      }
    ]
  }'
```

### 4.3 查询召回（`/rag/query`，默认 top_k=3）

```bash
curl -X POST "http://127.0.0.1:8080/rag/query" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "JT/T 809 协议默认通信端口和心跳周期是多少？"
  }'
```

### 4.4 查询召回（自定义 top_k 与 filters）

```bash
curl -X POST "http://127.0.0.1:8080/rag/query" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "OBD车辆故障码上报对应的报文ID是什么？",
    "top_k": 5,
    "filters": {
      "category": "policy"
    }
  }'
```

## 5. 常见问题排查

### 5.1 启动时报错：缺少 API_KEY_ALI

现象：服务启动失败，并提示缺少必要环境变量。  
处理：重新执行：

```bash
export API_KEY_ALI="你的阿里云百炼API Key"
```

### 5.2 查询时报错：INDEX_NOT_READY

现象：调用 `/rag/query` 返回 `400`，错误码 `INDEX_NOT_READY`。  
处理：先调用一次 `/rag/index` 完成文档入库，再执行查询。

### 5.3 参数校验失败（422）

常见原因：

- `documents` 为空数组
- `doc_id` 或 `file_path` 为空白
- `file_path` 不是 `.pdf` 文件
- `file_path` 指向的本地文件不存在
- `query` 为空白
- `top_k` 超出 `1-20`

请按接口模型要求修正请求体后重试。
